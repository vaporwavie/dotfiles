01

The source approach

The source is a "Picking the right models for workflows and subagents" section from another engineer's CLAUDE.md. Its core moves:

Rank every reachable model on three axes

cost (what you actually pay, not list price), intelligence (how hard a problem it takes unsupervised), taste (UI/UX, code quality, API design, copy). gpt-5.5 scores 9/8/5 — nearly free and smart, mediocre taste.

Defaults, not limits — standing permission to escalate

If a cheaper model's output misses the bar, rerun with a smarter one without asking. Judge output, not price tag. For anything that ships: intelligence > taste > cost.

Route by shape of work

Bulk/mechanical → gpt-5.5. User-facing → taste ≥ 7. Reviews → Fable/Opus plus optionally gpt-5.5 as an independent perspective. Token-hungry work (computer use, codebase analysis) never runs in the orchestrator's own context — delegate, report back.

Reach gpt-5.5 through the Codex CLI

codex exec / codex exec review; inside workflows (whose model param only takes Claude models) a thin Sonnet wrapper agent writes a self-contained codex prompt, runs it via Bash, returns the last message.

The author's claimed outcome: PR discard rate went from roughly 50% to zero once orchestration and execution were split across models this way.

Gaps & tuning knobs

GLM path: restored. The missing keychain item was re-added post-assessment and the profile verified live (security add-generic-password -a luiznickel -s codex-baseten-api-key -w '<key>' is the recovery command if it's ever lost again).

Fireworks: killed. It was doubly dead (no provider definition, no key) and redundant — both of its models are served through Baseten. fireworks.config.toml and its catalog are deleted, and the cx wrapper is repointed at the Baseten world: cx bt [--kimi|--glm] (Kimi boots by default, /model switches live in the TUI; fw still works as a legacy alias). Both routes verified via the session banner.

Scores are opinions. The table numbers are seeded from the source plus local judgment; tune them as experience accumulates — they're defaults the rules already grant permission to override.

Reasoning effort per delegation. The config default is medium; for hard delegated problems add -c model_reasoning_effort=high (or xhigh) to a single codex exec call instead of editing config.

Ephemeral by default. --ephemeral keeps delegated one-shots out of ~/.codex/sessions; drop it when you want codex exec resume to be able to continue the delegated thread.
